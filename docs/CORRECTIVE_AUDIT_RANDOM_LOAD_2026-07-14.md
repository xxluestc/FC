# 随机负载与在线退化链纠错审计

更新时间：2026-07-14

## 当前结论

此前`data/results/simplified_random_load_comparison`及Fig. 32作废。它只能证明代码可以完成
一次状态转移，不能证明随机动态负载、三堆N+1功率调控或退化感知方法已经跑通，也不能用于
比较策略优劣。

本次纠错后，只保留一个已经通过的基础事实：执行动作能够按
`action -> delta D -> D_after -> theta(D_after) -> IV -> voltage/power`
顺序在线写回世界模型。其退化率口径仍待修正。

## 为什么旧实验作废

1. **系统容量尺度错误。** 旧实验把四档需求设为
   `16.57/23.43/33.14/40.00 kW`，而当前LZW单堆在370 A时已经能够输出约
   `43.73 kW`。最高需求一只堆就能承担，三堆N+1失去测试意义。
2. **经验随机组没有动态。** 10个180 s配对种子合计1800 s中没有一次状态跳转，不能称为
   随机动态负载。
3. **时间基准没有文献依据。** Zuo的快/慢转移矩阵和四档状态有直接来源，但30 s是本项目
   后加假设。Zuo式(A.4)、快矩阵自保持概率0.1及文中400 s期望事件时长共同对应约360 s
   的Markov步长。
4. **“两堆运行”约束存在漏洞。** 当前动作空间允许一只堆正电流、另一只堆0 A通电待机，
   后者仍被计入在线堆，因此并未保证两只堆都实际承担负载。
5. **退化率分辨率不足。** 25--270 A全部使用`0.002%/h`；启动为`0.00196%/次`，
   变载为`5.93e-5%/次`，停机和爬坡代价为0。旧策略差异主要受事件常数和累计性能损失项
   驱动，不能解释为负载幅值退化优化。

## 40 kW和七天数据的准确含义

原始CSV字段顺序为“燃料电池状态、目标功率、DCDC输入电压、DCDC输入电流”。因此
`目标功率`可作为燃料电池控制器目标命令使用，但缺少厂家字段字典，不能称为铭牌额定功率或
实测净功率。全年运行样本`p99=40 kW`表示99%的运行样本不超过40 kW，不是均值。

刘占伟确实处理过这七天CSV：`data_pi_handle.m`读取半个月目录下CSV的第4、5、11--21列，
后续`data_process_new.m`和`sort_merge_new_update.m`用DCDC输入电压/电流提取启停、稳态电流档
和变载事件，最终得到`data_mark.mat`的203条事件记录。因此七天数据保留为刘占伟工况提取链
的来源证据，但不再直接充当三堆系统随机负载。

## Zuo四档如何映射到当前三堆系统

Zuo式(A.5)/(A.6)直接给出`[2.9, 4.1, 5.8, 7.0]`，Table 1给出单堆最大功率密度
`3.869 W/cm2`。两只工作堆的总上限为`7.738 W/cm2`，最高需求7.0保留约9.54%容量余量。

按当前健康LZW单堆370 A功率`43.7255 kW`等比例映射，四档系统需求应为：

| Zuo状态 | 当前系统需求 |
|---:|---:|
| 2.9 | 32.774 kW |
| 4.1 | 46.336 kW |
| 5.8 | 65.549 kW |
| 7.0 | 79.111 kW |

最高档低于任意两只初始仿真堆的合计最大功率，同时高于任意单堆最大功率，符合2-of-3
适度超容量系统的基本含义。下一轮必须要求两只工作堆都有正电流，不能用0 A待机凑在线数。

## G盘电压骤降段

匹配电流和冷却液温度后，实际日期2026-01-30至2026-02-06出现可恢复的异常电压低谷，
2026-02-07起恢复。后续使用G盘长期数据时，先排除区间
`[2026-01-30 00:00:00, 2026-02-07 00:00:00)`；原始文件不删除。拼接后的数组必须保留
`source_segment_id`和缺口边界，负载跳转、变载次数和持续时间不能跨缺口计算。
排除区间已固化在`configs/21ube0022_data_exclusions.json`。

## 在线健康链目前到底做了什么

累计状态`D`是文献暴露系数形成的percent-like工程损伤指标，不是真实SOH。当前参考终点为
`D_ref=9.3530285%`。三个旧测试堆的初始值为`0.1/0.4/0.8 D_ref`，即
`0.9353/3.7412/7.4824%`；异质性因子`1.00/1.05/1.10`只是仿真假设，不是三只实堆辨识结果。

每一步先根据实际动作计算：

`delta D = load + shift + start/stop + ramp + natural`

再更新`D_after=D_before+delta D`。随后使用

`theta_j(D)=theta_start,j + (theta_end,j-theta_start,j)*(D/D_ref)^p_j`

得到`[i0, ih, R]`，送入LZW单电池IV方程，最后计算
`P_stack=I*V_cell*170/1000`。世界模型在
`src/fc_power/world_model/mechanistic.py::MechanisticMultiStackWorldModel.step`中执行整条链。

Fig. 33使用固定动作序列直接审计该过程。540 s内`D`从`3.741211%`增至`3.743785%`，
`theta`和固定270 A下的电压/功率同步变化；但总增量`0.002574%`中单次启动已占
`0.00196%`，图中也直接显示25--270 A稳态退化率完全相同。该图是故障暴露图，不是方法效果图。

## 接下来只按三个验收门推进

1. **负载与容量门：** 使用约32.8--79.1 kW四档、Zuo原矩阵和可审计时间步；取消零跳转
   经验场景；保证最高档单堆不可达、任意两堆可达、两只工作堆均为正电流。
2. **退化链门：** 先把负载幅值率改成有来源、至少能区分低/中/高负载的简化函数；启动、
   变载和停机的定义单独列出，做量级敏感性，不再让一个事件常数支配短时结果。
3. **控制器门：** 枚举仅作为离散单步全局最优基线。链路通过后，再比较事件级连续优化或
   随机MPC；在未来功率预测冻结的前提下，使用Markov状态和期望事件时长，而不是偷看未来。

三个门通过前，不恢复Fig. 32，不报告策略改善百分比，也不进入TD-MPC、Dreamer或锂电池扩展。
