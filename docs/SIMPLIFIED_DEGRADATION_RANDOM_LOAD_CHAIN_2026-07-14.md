# 简化退化随机负载链路

更新时间：2026-07-14

## 这轮到底要解决什么

这轮不追求把退化系数辨识得很准，也不调权重追求更大的数字。目标只有一个：确认随机动态负载进入
三堆N+1系统后，功率分配、动作执行、在线退化更新和结果统计能从头到尾连续跑通，并且可以公平比较
“控制器考虑健康”和“控制器不考虑健康”。

## 链路怎么走

1. 生成三类随机负载：现有40 kW口径下的实车1秒经验Markov负载、Zuo慢变30秒压力负载、
   Zuo快变30秒压力负载。这里不是直接回放一条实车功率曲线。
2. 三个模拟电堆共用LZW单堆模型，再用初始老化程度和名义异质性因子构造不同堆身份；任何时刻
   恰好两堆运行、一堆休息，不含锂电池。
3. 比较Average、Zuo式DC-average、Rotating、Instant-no-health和Instant-health五种策略。
4. 每秒执行离散电流动作，世界模型按实际动作更新损伤`D`，再经过`D -> theta -> IV/功率`影响
   下一步状态。没有使用未来需求功率。
5. 所有策略面对同一负载种子、同一初始健康和同一退化过程。健康消融只关闭规划目标中的退化与
   性能损失项，不关闭被控对象的老化。

## 退化目前简化到什么程度

当前保留确定性的动作驱动工程代理，包括连续运行、启停和变载损伤。它能表达“做了什么动作，就产生
相应健康变化”，足以检查闭环和策略差异；它不能解释为真实车辆上已经标定好的物理退化率。

LZW总体暴露筛查没有识别出稳定的负载分辨退化规律：负载项相对运行时钟没有额外预测增益，而且
运行时长与电荷暴露高度相关。因此LZW数据目前只保留健康终点和总体趋势作用，不拿来硬拟合启停、
高低负载或变载系数。Zuo只提供N+1系统思路和随机负载矩阵，不提供本项目的退化系数。

短时Gamma采样、锂电池、未来负载预测和实车健康观测校正均未进入本轮。

## 正式实验和验收

- 规模：3类随机负载 x 5种策略 x 10个同种子，共150次运行；每次180秒。
- 初始健康：三堆损伤分别位于LZW参考终点的10%、40%和80%。
- 名义异质性：`1.00/1.05/1.10`。
- 150/150运行步数完整；每一步健康状态都发生在线更新。
- 150/150始终恰好两堆在线。
- 约束违规、跟踪超差、动作裁剪和安全接管均为0。

这说明“随机负载 -> 功率调控 -> 实际动作 -> 在线退化 -> 下一步性能 -> 配对统计”已经跑通。

## 现在能看到什么结果

Instant-health相对Instant-no-health的最大单堆损伤增量变化为：

| 随机负载 | 最大单堆损伤 | 单位电量氢耗 | 跟踪MAE绝对变化 |
|---|---:|---:|---:|
| 实车1秒经验Markov | -0.91% | -0.10% | +0.023 kW |
| Zuo慢变 | -2.21% | -0.10% | +0.022 kW |
| Zuo快变 | -2.22% | -0.12% | +0.016 kW |

10个种子中，健康目标在最大单堆损伤上分别为“2胜8平、5胜5平、4胜6平”，没有出现更差的种子。
实车经验负载的配对区间仍跨0；两个Zuo压力场景的最大单堆损伤区间刚好低于0。这个结果说明健康项
已经进入决策并能改变部分动作，但影响还弱，不能据此声称方法已经显著延寿。

DC-average每30秒强制轮换运行堆，三类负载下最大单堆损伤比Average高约143%到172%，主要来自
12次切换，而Average通常只有初始化的2次启动。它能改善三堆暴露均衡，却被当前启停代理的代价压过。
Rotating介于两者之间。这个现象先原样保留，后续优化时再判断是调度逻辑问题还是事件代价口径问题。

## 当前明确存在的问题

1. 退化模型是工程代理，不是实车辨识率；短时损伤数值只能做相对比较。
2. 180秒只验证闭环和动作差异，不能回答寿命或EOL问题。
3. 最低16.57 kW系统负载在原25 A/90 A离散电流之间没有可跟踪点。本轮加入60 A插值动作，
   它来自既有IV曲线插值，不是LZW原始事件档位。
4. 健康与非健康目标在一半以上种子中选择相同结果，说明当前健康项的决策分辨率不足。
5. DC-average对启停损伤系数很敏感；在系数没有实车或台架证据前，只能当对照，不能据此否定轮换。
6. 实车经验Markov负载仍是随机生成序列，不等于未参与建模的连续真实轨迹回放。
7. 40 kW是已有实车数据支持的经验归一化参考，不是已经确认的铭牌额定净功率。

## 本轮结论

链路已经跑通，结果不是空的：退化随动作在线更新，健康目标在部分种子中改变了分配，并出现小幅
损伤改善。与此同时，效果偏弱、短时尺度有限、60 A动作是插值点、事件损伤缺少物理标定，这四个
问题都已暴露出来。下一阶段才进入效果优化，本轮不通过调权把这些问题遮住。
