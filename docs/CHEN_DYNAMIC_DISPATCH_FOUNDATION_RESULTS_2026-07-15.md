# 陈鹏效率曲线上的三堆动态功率分配基础结果

更新时间：2026-07-15

## 1. 先说结论

这条效率优化路线已经跑通了一个可信的基础闭环：

```text
随机动态净功率需求
-> 枚举当前可运行的单堆/双堆组合
-> 在每个组合内求全局最优净功率分配
-> 在线选择是否换堆
-> 执行动作并逐秒累计氢耗、效率和启停变化
```

在没有使用未来功率预测的条件下，在线 break-even 滞回策略在独立留出种子上，相对每秒只追求最低氢耗的 instantaneous 策略：

- 平均减少 `46.59% ± 11.53%` 的电堆状态变化；
- 平均增加 `0.190% ± 0.045%` 的氢耗；
- 10 个留出种子中，10 个都减少了切换；
- 所有策略、所有时间步的净功率平衡误差均小于 `1e-8 kW`。

这证明的是“氢耗与切换之间存在稳定、可控制的权衡”，不是已经证明真实车辆寿命提高。当前没有使用退化模型、锂电池、未来需求预测或真实爬坡率。

## 2. 输入功率和三条曲线从哪里来

### 2.1 三个虚拟电堆

陈鹏第四章用相同技术路线、不同片数构造了三个虚拟电堆：`270/300/330` 片。曲线原始横轴是电堆毛功率，效率则是扣除空压机功耗后的 LHV 系统效率。本项目已经将横轴统一重建为净功率。

| 电堆 | 曲线支持的净功率域 (kW) | LHV 峰值效率 | 峰值效率对应净功率 (kW) |
|---|---:|---:|---:|
| stack 1, 270 cells | 5.664-54.263 | 48.074% | 23.169 |
| stack 2, 300 cells | 6.635-60.079 | 48.635% | 26.044 |
| stack 3, 330 cells | 7.607-65.807 | 49.158% | 24.820 |

陈鹏论文还给出一个 `340` 片、`406 cm2` 有效面积、`120 kW` 额定电堆平台。按相同有效面积和片数线性缩放，只能得到 `95.29/105.88/116.47 kW` 的毛功率铭牌等效推断。该推断没有用于调度，也没有把效率曲线外推到铭牌功率。

当前运行下限采用曲线第一个有效采样点，即 `0.1 A/cm2` 对应的净功率。它是保守的插值边界，不是已经验证的物理最小稳定功率。关闭状态单独建模为 `OFF`。

### 2.2 随机动态负载

四个功率中心完全由陈鹏曲线推导，不使用 Zuo 的 `2.9/4.1/5.8/7.0` 幅值：

| 状态 | 构造方式 | 中心功率 (kW net) |
|---|---|---:|
| 低负载 | 三条曲线峰值效率功率的中位数 | 24.820 |
| 双堆高效负载 | 两个最高峰值效率功率之和 | 50.864 |
| 高负载 | N+1 两堆最大净功率的 75% | 94.414 |
| 备用边界负载 | N+1 两堆最大净功率的 90% | 113.297 |

N+1 两堆最大净功率为 `125.885 kW`。Zuo fast 矩阵只决定四类事件之间如何跳转；每段驻留 `8-25 s`，段内目标围绕中心随机变化 `±3%`。这两个数是为了检验在线切换的工程压力设置，不冒充实车驻留或爬坡规律。

开发集使用种子 `2026-2035`，留出集使用 `3026-3035`，每条轨迹 `900 s`。算法和权重在查看留出结果前冻结。

## 3. 每一秒到底怎么分配

给定当前净功率需求 `Pdem,t`，先枚举：

```text
单堆：{1}, {2}, {3}
双堆：{1,2}, {1,3}, {2,3}
```

正常最多两堆运行。对每个当前可行组合 `m`，求解：

```text
min  sum_j Pchem,LHV,j(Pj)
s.t. sum_j Pj = Pdem,t
     Pj 位于该堆净功率插值域内
     j 不属于 m 时 Pj = 0
```

两堆时只有一个独立功率变量。每条化学输入曲线采用分段线性插值，因此总目标也是分段线性的；全局最小值必定位于可行区间端点或任一曲线断点。当前求解器完整枚举这些点，不依赖黑箱优化器，也不是粗网格近似。

得到每个组合的最低当前氢耗后，外层策略决定是否切换组合。执行后记录每堆净功率、毛功率、效率、电流密度、系统氢耗和开关状态，再进入下一秒。

## 4. 比较了哪些策略

| 策略 | 当前实现含义 |
|---|---|
| Average | 固定选择容量最大的两堆，尽量平均分配净功率 |
| Daisy chain | 按峰值效率排序，先尽量加载一堆，再加入第二堆 |
| Instantaneous | 每秒选择当前氢耗最低的组合和组合内分配 |
| Sticky | 只要上一组合仍可行就不换，组合内仍重新优化功率 |
| One-step greedy | 当前氢耗加本步切换代价最小 |
| Break-even hysteresis | 累计换堆可节省的氢耗，足以偿还切换代价后才换 |
| Offline DP | 使用整条未来需求的事后全局最优，只作为下界 |

切换代价用等效氢耗表示。基准值 `0.119138 g/stack-state-change` 来自低负载下“双堆维持”相对“单堆高效运行”的额外氢耗，在中位驻留 `16.5 s` 内做一次停堆和一次未来重启的 break-even 换算。它不是退化系数，也不是真实启停维修成本。

可选停堆采用两次状态变化阈值，因为一次完整往返包含当前停堆和未来重启。这个解释已单独做了对称阈值复核：对称阈值更省少量氢，但在开发/留出集分别仍有 `20.4/19.8` 次状态变化；往返阈值降至 `12.2/11.6` 次，综合目标也更低。

## 5. 实际结果

### 5.1 主权重下的留出结果

| 策略 | 平均氢耗 (g/900 s) | LHV 能量加权效率 | 平均状态变化 | 相对 Instantaneous 氢耗 |
|---|---:|---:|---:|---:|
| Average | 1198.401 | 44.356% | 2.0 | +0.562% |
| Daisy chain | 1373.988 | 38.700% | 28.9 | +15.266% |
| Instantaneous | 1191.729 | 44.605% | 20.8 | 0 |
| Sticky | 1195.381 | 44.466% | 2.0 | +0.313% |
| One-step greedy | 1195.381 | 44.466% | 2.0 | +0.313% |
| Break-even hysteresis | 1193.937 | 44.521% | 11.6 | +0.190% |
| Offline DP | 1192.525 | 44.576% | 11.6 | +0.067% |

在当前基准权重下，online hysteresis 的综合目标仍比 instantaneous 高 `0.098% ± 0.056%`，所以不能宣称它在任意切换估值下都更优。

当切换权重提高到基准值的 `2x` 时，留出集在线滞回平均只剩 `2.8` 次状态变化，综合目标相对 instantaneous 改善 `0.144% ± 0.038%`；在 `4x` 时改善 `0.519% ± 0.085%`。开发集在同两点分别改善 `0.141%` 和 `0.540%`，转折方向一致。

One-step greedy 与 Sticky 在主权重下行为完全相同，不能把它们包装成两个独立有效结果。Daisy chain 在当前异构曲线上氢耗和切换都更差，是有价值的负结果。

## 6. 当前能说什么，不能说什么

现在可以说：

- 异构三堆净功率/LHV 效率数据口径已经统一；
- 单堆/双堆组合内分配是可审计的全局最优；
- 无未来需求时，在线滞回能稳定减少不必要切换；
- 开发集与独立留出集复现了同一权衡；
- offline DP 是同一目标下的事后下界。

现在不能说：

- 已经提高真实车辆寿命或降低真实启停退化；
- 曲线端点就是厂家最小/额定功率；
- 已满足真实爬坡和最小驻留时间；
- 已完成 N+1 故障容错验证；
- offline DP 或 AAAI 预测算法可在线直接使用；
- 当前 break-even hysteresis 等同 Borodin MTS `A_f` 或继承 NeurIPS 理论界。

## 7. 文献如何组合，而不是只换算法名字

| 文献 | 在当前框架中的实际作用 |
|---|---|
| 陈鹏学位论文 | 三条异构效率曲线、辅助功耗和基础分配问题 |
| Igourzal et al., ECM 2024, DOI 10.1016/j.enconman.2024.118630 | 多堆组合枚举、异构性、效率和耐久性的系统级权衡 |
| Haubensak et al., ECM 2026, DOI 10.1016/j.enconman.2025.120598 | 启停离散变量、氢耗-启停 Pareto 实验和结果图结构 |
| Zhang et al., NeurIPS 2021, *Revisiting Smoothed Online Learning* | 当前 hitting cost 加 switching cost 的在线问题结构 |
| Lin et al., AAAI 2023, *Smoothed Online Combinatorial Optimization Using Imperfect Predictions* | 未来加入预测不确定性和动态窗口；当前明确不用 |
| Chi et al., Automatica 2025, *Online mixed discrete and continuous optimization* | 离散组合加连续分配的形式化参考；其 bandit 探索不适合当前已知曲线 |

NeurIPS 2021 的理论依赖凸性等条件，当前离散组合加分段线性曲线不直接满足其完整假设。因此这里只采用问题结构，不能照搬竞争比或动态遗憾结论。

## 8. 独立复核与下一步

本地 Claude Code/DeepSeek 做了两轮只读复核。代码复核确认：分段线性全局求解、氢耗单位、offline DP 下界、初始 OFF 切换口径和开发/留出隔离均正确。科研复核指出三个硬缺口：物理功率边界、真实爬坡率和故障注入。

下一步先做故障注入，而不是直接假设一个厂家爬坡率。原因是当前已经把系统称为三堆 N+1，必须先证明任一堆故障退出后，剩余两堆可以重新分配并保持功率平衡。爬坡率在没有厂家或台架依据前，只能做参数敏感性，不能写成真实约束。
