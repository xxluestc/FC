## 1. 当前研究问题

当前首先解决三堆PEMFC系统内部的运行堆选择和功率分配：

```text
当前已知需求功率
+ 三个电堆各自的D/theta/IV状态、电流、开关和驻留状态
        ↓
候选动作[I1, I2, I3, on1, on2, on3]
        ↓
三堆N+1硬约束：最多两个堆在线，一个堆轮换休息
        ↓
实际执行动作驱动各堆确定性健康更新
D_i(k+1) → theta_i(k+1) → V_i/P_i(k+1)
        ↓
满足燃料电池侧目标功率，同时权衡氢耗、退化、性能损失、启停和变载
```

近期不把锂电池、SOC恢复和未来需求预测放入主决策问题。它们是后续外层优化，
不能阻塞燃料电池内层基础闭环。

## 2. 冻结边界

除非形成新的书面决策，以下口径保持不变：

| 项目 | 当前口径 |
|---|---|
| 系统结构 | 三堆适度超容量N+1，同一时刻最多两个堆在线 |
| 当前决策层 | 多个燃料电池之间的运行堆选择和功率分配 |
| 当前负载信息 | 只使用当前需求，不做未来需求预测 |
| 锂电池 | 暂不进入当前内层决策；保留未来外层接口 |
| 健康更新顺序 | 确定性动作更新 → Gamma随机退化 → 实车观测校正 |
| 单堆负载字段 | `liu_vehicle_canonical_1s.csv:fc_input_power_kw` |
| 标定/留出边界 | segment 0-21标定；segment 22-45完整留出 |
| 退化代理口径 | `C_deg(I|theta)`是当前健康下性能损失代理，不是材料退化直接观测 |
| Gamma口径 | 均值由动作/事件驱动；方差/CV目前是敏感性假设 |
| 文献系数 | Ghaderi/Pei系数必须标来源，不称为LZW数据因果辨识 |
| 高级算法 | 基础门槛通过前不训练Dreamer/EAWM/TD-MPC类模型 |

研究判断按以下顺序取证：

1. 本地原始数据、作者MAT结果链和可复现代码输出。

3. 仓库内审计报告、单元测试和冻结实验结果。

### G1：三堆确定性健康基础闭环

状态：已通过。

已验收：

- 动作枚举和世界模型均强制最多两个堆在线；
- 每堆只根据自身实际执行动作更新D；
- D单调更新后映射theta，再由统一IV模型更新电压和可用功率；
- 关机休息堆连续退化为零，0 A带电怠速与完全停机保持区分；
- 规划动作与实际执行动作、确定性增量与随机增量分别记录；
- 增加三堆轮休、长期停机零退化、变载计数一致性回归测试。

出口证据：三堆运行15步后执行轮换，休息堆保持零退化，轮换后各堆按实际动作更新；
`D→theta`逐堆一致，世界模型与暴露提取器的变载计数一致，N+1硬约束通过。

### G2：燃料电池内层功率接口

状态：已通过基础出口验收。

世界模型现在显式支持历史`battery`和新增`fc_only`两种功率接口。`fc_only`必须显式给出
跟踪容差，电池功率固定为0、SOC冻结，跟踪误差进入独立成本和硬约束；旧电池模式保持默认。

已完成：

- 定义不依赖电池/SOC的燃料电池侧目标功率和跟踪误差；
- 保留离散电流、驻留、爬坡、启停和最多两堆在线约束；
- 明确离散档位下允许的功率跟踪容差或局部连续修正方式；
- 旧混合系统模式保持兼容，供未来外层氢电分配复用；
- FC-only评价不再使用SOC修正氢耗、电池吞吐或SOC恢复尾段。

当前8档电流、两堆在线的健康聚合功率最大相邻间隔约10.60 kW，半间隔约5.30 kW；
5.5 kW仅作为由当前动作网格推导的候选容差，不是通用物理常数。Average、Rotating、Instant、
Beam、安全投影和统一测试床均已通过FC-only单元/短轨迹测试，SOC恢复直接调用会被拒绝。

实车1秒基线与Zuo快/慢独立压力场景的三条120步确定性轨迹均零违规，最大跟踪误差
4.018 kW；四状态、四策略静态动作检查最大误差同为4.018 kW，均低于5.5 kW候选容差。
该容差仍需在后续长轨迹和健康演化中复核，不能通过任意放宽掩盖不可行区间。

### G3：Zuo-实车随机动态负载

状态：已通过基础出口验收。

已完成：

- 核对Zuo 2024三堆两运行、无电池、四状态Markov结构；
- 实车单堆数据按最大时间间隔切成标定段与完整留出段；
- Zuo相对状态在标定段的占比审计；
- 1/5/10/30/60秒转移矩阵和segment级bootstrap 95%区间。
- 新增分段常值四状态生成器，显式输入转移矩阵、决策间隔、初始分布、容量参考和种子；
- 审计1/5/10/30/60秒经验矩阵的等效状态变化率，确认下采样会漏掉中间变化；
- 取消不同时间基准的经验矩阵与Zuo矩阵直接凸组合；
- 选择实车1秒经验矩阵作为主标定场景，保持原生1秒时间基准；
- 将Zuo慢变/快变矩阵保留为独立压力场景，30秒转移步明确标为工程候选；
- 负载上限映射为两台健康堆容量的95%，正需求时强制恰好两堆在线；
- 0/2/5/10/15%容量余量在健康、异质和后期三类状态下完成动作覆盖扫描；
- 三条120步FC-only确定性轨迹零违规，未使用留出段功率分布。

后续验证：

- 在确定性多策略比较中复核5%容量余量和5.5 kW容差的长轨迹覆盖；
- 随机负载用于开发，segment 22-45仅用于最终连续真实回放。

出口条件：状态、转移、时间尺度、容量映射和随机种子全部可追溯。

### G4：确定性控制基线和公平评价

- 可以多个只读代理并行，并按任务需要使用`--effort max`。
