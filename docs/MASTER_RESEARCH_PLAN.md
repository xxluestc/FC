## 1. 当前研究问题

当前首先解决三堆PEMFC系统内部的运行堆选择和功率分配：

```text
当前已知需求功率
+ 三个电堆各自的D/theta/IV状态、电流、开关和驻留状态
        ↓
候选动作[I1, I2, I3, on1, on2, on3]
        ↓
三堆N+1硬约束：最多两个堆在线，一个堆轮换休息
        ↓
实际执行动作驱动各堆确定性健康更新
D_i(k+1) → theta_i(k+1) → V_i/P_i(k+1)
        ↓
满足燃料电池侧目标功率，同时权衡氢耗、退化、性能损失、启停和变载
```

近期不把锂电池、SOC恢复和未来需求预测放入主决策问题。它们是后续外层优化，
不能阻塞燃料电池内层基础闭环。

## 2. 冻结边界

除非形成新的书面决策，以下口径保持不变：

| 项目 | 当前口径 |

| 当前决策层 | 多个燃料电池之间的运行堆选择和功率分配 |
| 当前负载信息 | 只使用当前需求，不做未来需求预测 |
| 锂电池 | 暂不进入当前内层决策；保留未来外层接口 |
| 健康更新顺序 | 确定性动作更新 → Gamma随机退化 → 实车观测校正 |
| 单堆负载字段 | `liu_vehicle_canonical_1s.csv:fc_input_power_kw` |
| 标定/留出边界 | segment 0-21标定；segment 22-45完整留出 |
| 退化代理口径 | `C_deg(I|theta)`是当前健康下性能损失代理，不是材料退化直接观测 |

| 文献系数 | Ghaderi/Pei系数必须标来源，不称为LZW数据因果辨识 |
| 高级算法 | 基础门槛通过前不训练Dreamer/EAWM/TD-MPC类模型 |

## 4. 阶段门槛

### G0：仓库与数据可复现

状态：已通过。

- 交接补丁完整应用；
- 关键数据可物化；
- 65项测试、baseline、compileall、`git diff --check`通过；
- 受保护数据文件未被修改。

### G1：三堆确定性健康基础闭环

状态：进行中，主体已具备。

必须完成：

- 动作枚举和世界模型均强制最多两个堆在线；
- 每堆只根据自身实际执行动作更新D；
- D单调更新后映射theta，再由统一IV模型更新电压和可用功率；
- 关机休息堆连续退化为零，0 A带电怠速与完全停机保持区分；
- 规划动作与实际执行动作、确定性增量与随机增量分别记录；
- 增加三堆轮休、长期停机零退化、变载计数一致性回归测试。

出口条件：三堆确定性轨迹逐步可审计，所有健康不变量和N+1约束通过。

### G2：燃料电池内层功率接口

状态：未完成，是当前代码的主要结构缺口。

当前世界模型仍令电池自动吸收`需求功率-电堆功率`残差。不能只把电池权重设为0，
因为电池充放功率和SOC仍会影响动作可行性。

必须完成：

- 定义不依赖电池/SOC的燃料电池侧目标功率和跟踪误差；
- 保留离散电流、驻留、爬坡、启停和最多两堆在线约束；
- 明确离散档位下允许的功率跟踪容差或局部连续修正方式；
- 旧混合系统模式保持兼容，供未来外层氢电分配复用；
- FC-only评价不再使用SOC修正氢耗、电池吞吐或SOC恢复尾段。

出口条件：无电池补偿时功率误差显式可见且受约束，控制器不能靠电池隐藏不可行动作。

### G3：Zuo-实车随机动态负载

状态：已完成时间隔离和多尺度审计，最终生成器未完成。

已完成：

- 核对Zuo 2024三堆两运行、无电池、四状态Markov结构；
- 实车单堆数据按最大时间间隔切成标定段与完整留出段；
- Zuo相对状态在标定段的占比审计；
- 1/5/10/30/60秒转移矩阵和segment级bootstrap 95%区间。

必须完成：

- 选择并记录Markov决策时间尺度，不把论文未明确的时间单位写成事实；
- 透明组合Zuo快/慢矩阵先验与实车标定统计；
- 将归一化单堆负载映射到两台在线堆的健康相关可用功率；
- 随机负载用于开发，segment 22-45仅用于最终连续真实回放。

出口条件：状态、转移、时间尺度、容量映射和随机种子全部可追溯。

### G4：确定性控制基线和公平评价

状态：等待G1-G3。

比较Average、Rotating、Instant-health和Beam-health。核心指标为燃料电池功率跟踪误差、
总氢耗、各堆退化与离散度、性能损失、启停/变载、在线/轮休时长、连续在线/停机时长、
计算时间和候选动作数。FC-only主实验不使用SOC或电池指标。

### G5：Gamma随机退化与统计评价

状态：基础代码已有，暂不作为默认主实验。G4通过后使用同负载、同健康随机种子的配对实验，
报告差值、相对变化、95%置信区间和改善率。

### G6：实车观测校正

状态：等待MAT观测链。先定义`HealthObserver.predict/correct`接口和合成观测测试；
获得完整变量后再复现UKF/PF posterior。在此之前只能称为动作驱动预测态。

### G7：基础模型消融与优化

进入条件：G4-G6相应门槛通过。包括退化权重、性能损失项、事件系数、Markov时间尺度、
动作网格、Beam宽度和时域等。使用Pareto前沿和多种子稳定性选参数。

### G8：后续外层和高级算法

状态：冻结。按顺序考虑锂电池外层、未来需求预测、机理残差世界模型、Dreamer/EAWM，
连续动作条件成熟时再评估TD-MPC类方法。
