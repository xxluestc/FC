## 2. 最终研究目标

构建面向多堆 PEMFC/动力电池系统的退化感知预测功率调控框架，使控制器能够：

```text
估计各堆当前健康状态
→ 在候选功率动作下预测未来需求、SOC、氢耗和健康变化
→ 进行多堆与电池之间的滚动功率分配
→ 执行第一步动作
→ 利用新观测修正健康状态与模型
→ 进入下一轮在线决策
```

长期算法名称暂定：

```text
EADWM-MPC / EAWM-Dreamer-MPC
事件感知退化世界模型预测功率调控
```

最终目标不是仅降低一个加权目标值，而是在功率平衡和安全约束下，公平比较氢耗、燃料电池退化、电池负担、SOC维持、功率波动、启停和多堆健康均衡。

## 3. 当前已经具备的基础

### 3.1 数据与预测链

- 刘占伟 21UBE0022 半月整车数据已整理成 1 s canonical 表，共 211,630 行、46 个连续运行片段。
- 当前预测目标为整车需求功率 `P_dem`，而不是由控制策略决定的燃料电池输出功率。
- 已有 State-Aware Direct Power Prediction：使用历史车速、加速度、需求功率、SOC、停车/路线等因果状态，分别预测 H=1/3/5/10 s。
- 当前冻结模型为 XGBoost；H=5 测试逐点 RMSE 约 22.14 kW，H=10 约 33.55 kW。
- 车辆动力学模型保留为物理可解释基线，不和直接功率预测混称。

### 3.2 单堆氢电分配链

- 当前动作是单个燃料电池的离散电流/功率档位，电池通过 `P_bat=P_dem-P_fc` 补足功率。
- 已有 Rint/SOC 电池模型、法拉第理论氢耗模型和 Beam-search MPC。
- 已对比 Instant、Constant、Perfect-preview 和 Predicted MPC。
- `scripts/run_baseline.py --mode check` 可在不依赖原始大文件时完成轻量验证。

### 3.3 退化代理链

- 刘占伟 UKF-PF 提供 6104 个稳定事件上的健康参数轨迹 `theta(k)=[i0, ih, R_ohm]`，现已包含逐事件ID、canonical行号和原始索引。
- 上游累计事件表、theta物理参数、IV模型和当前点损失表已经完成对齐审计。
- 已实现动作耦合Gamma健康状态和动态性能损失代理。当前代理的准确含义是：


## 4. 当前完整任务分解

用户提出的三项任务是主线，但要形成可验证系统，还必须补齐观测、多堆环境、安全约束和公平评价。

### 任务 A：动作耦合的在线健康状态更新

目标：让各堆健康状态随实际决策变化，而不是始终使用固定 `theta_late`。

第一版采用可解释的 Gamma 累计退化状态：

```text
D_i(k+1) = D_i(k) + DeltaD_i(k)
DeltaD_i(k) ~ Gamma(alpha_i(action,event,dt), beta_i)
theta_i(k) = g_i(D_i(k))
```

其中均值退化强度至少区分：

- 负载幅值/电流档位；
- 负载变化；
- 启停；
- 怠速/低功率；
- 停机休息；
- 堆间随机异质性。

第一版不声称从现有数据精确辨识了所有因果系数。均值结构参考 Pei/Ghaderi/Zuo，Gamma 过程负责不可逆正增量和随机性；刘占伟 `theta(k)` 用于健康轨迹尺度映射和合理性验证。

还必须增加观测修正接口：

```text
prediction:  D_i^- / theta_i^-
observation: current, voltage, temperature, pressure ...
correction:  UKF/PF/FFRLS or a simplified calibrated observer
posterior:   D_i^+ / theta_i^+
```

没有新观测时只能做状态预测，不能宣称完成了在线观测修正。

### 任务 B：真正的多堆功率分配环境

目标：由单堆 `P_fc + P_bat` 扩展为：

```text
P_dem = sum(P_fc_i) + P_bat
u_k = [I_1, I_2, ..., I_n, on/off_1, ..., on/off_n]
```

环境状态至少包括：

- 当前需求功率与短时预测；
- SOC、电池功率与上一动作；
- 每堆 `D_i/theta_i/SOH_i`；
- 每堆电流、开关状态、驻留时间和启停计数；
- 怠速、高载、变载、制动、SOC紧张、健康失衡等事件。

约束至少包括：

- 瞬时功率平衡；
- 电池功率与SOC上下限；
- 电堆功率/电流范围；
- 最小开停机/档位驻留时间；
- Ramp与启停限制；
- 实验末端SOC等值。

必须先实现枚举/Beam-search 多堆基线，再替换为高级算法，否则无法判断高级算法的真实收益。

### 任务 C：未来需求功率预测优化

预测对象固定为整车需求功率 `P_dem`，因为燃料电池需求功率是控制决策结果。

后续按以下层次比较：

1. Persistence/Constant、RLS/AR和现有XGBoost基线；
2. 速度预测→车辆动力学换算的物理基线；
3. 工况/事件条件化的多专家预测器，重点处理制动、高功率和突变事件；
4. 机理预测 + 学习残差；
5. TCN/GRU/PatchTST等直接多步序列模型；
6. 分位数或场景式概率预测，将不确定性传入鲁棒/场景MPC。

预测模型不能只按总体RMSE选择。必须同时评价：

- MAE、RMSE及规范化指标；
- 未来窗口能量误差；
- 高功率和制动事件F1；
- 概率预测覆盖率/校准度；
- 相对Perfect-preview的控制决策后悔值；
- 对最终氢耗、退化、SOC和切换的下游收益。

当前最值得优先验证的是“事件条件化 + 概率多步预测”，因为现有主要误差来自缺少外部触发信息的突发制动/负载变化；仅继续增加普通历史统计特征，已有消融结果显示收益很小。

### 任务 D：模块化机理世界模型

在引入 Dreamer/TD-MPC/EAWM 前，先建立可调用、可测试的一步转移接口：

```text
next_state, cost, constraint_info = world_model.step(state, action, demand)
```

内部由以下模块组成：

- 电堆 `I,theta → V,P,H2,efficiency`；
- Gamma健康转移 `D,action,event → D_next/theta_next`；
- 电池 `SOC,P_bat → SOC_next`；
- 事件更新；
- 氢耗、FC退化、电池使用、SOC、波动和启停成本；
- 动作掩码和硬约束检查。

未来SOC、退化、健康状态和氢耗由该模型递推得到，而不是作为外部已知输入。

### 任务 E：状态观测、校准与可辨识性

需要明确区分：

- 状态：`D/theta/SOH`；
- 动作：各堆电流、功率和开关；
- 观测：电压、电流、温度、压力等传感量；
- 代理成本：`C_deg(action | state)`；
- 真实材料退化：当前数据无法直接完整观测。

必须完成单位、时间步长和尺度审计，避免把“每事件”“每秒”“每小时”的参数混用。任何暂用文献或仿真参数都写入配置并标注来源，不写死在算法内部。

### 任务 F：安全投影与公平实验协议

高级策略输出不能直接执行，必须经过：

```text
动作掩码 → 可行性检查 → 安全投影/最近可行动作 → 执行
```

所有策略使用相同的数据切分、模型、动作空间、SOC条件和可行域。最终主比较应满足硬末端SOC约束，例如 `|SOC_end-SOC_ref|<=0.001`；若临时不能满足，必须使用SOC等值补能指标并同时报告原始值。

实验至少包含：

- Average / Daisy-chain / Rotating / Instant；
- Constant/Predicted/Perfect-preview MPC；
- 退化不感知与退化感知；
- 确定性退化与Gamma随机退化；
- 单堆/双堆/三堆及健康异质性；
- 多工况、多随机种子、消融和敏感性分析。

### 任务 G：世界模型与高级算法升级

算法按下列顺序升级，不直接跳到最终模型：

#### Phase G1：机理/表格世界模型 + Beam-search MPC

用于证明状态、动作、健康转移、约束和评价闭环正确，是所有学习算法的对照组。

#### Phase G2：机理模型 + 学习残差

用 MLP/GRU/XGBoost 学习一步误差：

```text
real_next_state = mechanistic_next_state + learned_residual
```

先验证学习模块是否改善多步 rollout，而不是只看一步训练误差。

#### Phase G3：Dreamer-style RSSM + EAWM事件增强

Dreamer作为潜在状态世界模型和 imagined rollout 主体；EAWM增加事件预测、事件边界检测和事件感知表征，使模型关注启停、制动、高载、变载、SOC紧张和健康失衡等关键事件。

EAWM不是独立控制器，而是可以接入Dreamer等世界模型的表征增强模块。

#### Phase G4：规划与安全执行

保留MPC/Beam-search或安全投影层，确保学习策略不违反硬约束。可比较：

- Dreamer actor直接给出候选动作 + 安全投影；
- Dreamer世界模型内 imagined rollout + MPC选择；
- EAWM-Dreamer-MPC完整模型。

TD-MPC2作为重要参考和对照，但暂不作为当前离散档位环境的首选主算法。其原论文主要面向连续动作，并明确指出离散动作规划扩展仍是开放问题。如果后续把各堆功率放松为连续动作，或采用适合离散组合动作的搜索器，才进入直接TD-MPC式实现。

## 6. 各阶段最低验收条件

### P1健康更新

- 固定随机种子可复现；
- Gamma增量非负，健康状态不会无解释地逆向恢复；
- 高负载、变载和启停能够改变均值退化强度；
- 不同堆可配置异质性；
- 单位和时间步长明确；
- 确定性均值模式可用于调试。

### P2动态代理与修正

- `C_deg(I|theta_early/middle/late)`随健康阶段发生合理变化；
- 预测状态和观测修正状态分开记录；
- 无观测时不伪造posterior更新；
- 与现有late静态表可回归对齐。

### P3多堆闭环

- 每一步功率平衡误差在数值容差内；
- SOC、电池功率、驻留和动作约束零违规；
- 能复现Average/Rotating等简单策略；
- 老化堆能够在健康感知策略下获得更多休息或更低负载；
- 末端SOC公平。

### P4预测升级

- 时间顺序切分且H秒purge；
- 不使用未来泄漏字段；
- 至少在事件指标、概率校准或控制收益之一稳定超过当前XGBoost；
- 如果只降低总体RMSE但不改善控制，不升级默认模型。

### P5/P6学习世界模型

- 多步rollout误差不发散；
- 在未见运行片段上评价；
- 学习策略经过安全投影后零硬约束违规；
- 相对机理MPC的提升在多工况和多随机种子上存在，而不是单次偶然结果；
- 必须做事件模块、Gamma模块、latent模型和规划器消融。

### 2026-07-12：高级世界模型材料适配结论

- EAWM的核心是为基础世界模型增加自动事件生成、事件预测和Generic Event Segmentor，并通过辅助目标塑造潜在表示；它不是独立控制器。当前PEMFC事件已有物理定义，因此第一版应使用高载、制动、启停、变载、SOC紧张和健康失衡作为多模态事件标签，而不是照搬视觉AGMM。
- TD-MPC2原始问题定义和规划器面向连续动作，使用潜在动力学、奖励/Q预测、策略先验和采样式局部轨迹优化。当前离散多堆档位不能直接照搬其连续采样规划；可借用decoder-free latent dynamics与TD价值头，但动作选择仍需离散Beam或安全投影。
- 因此高级算法顺序保持不变：先做机理残差的一步/多步rollout基准，再做Dreamer-style潜在动力学，最后添加EAWM事件辅助头；任何学习策略都必须通过当前统一测试框架和安全执行层。
