## 2. 共识优点

1. 开发/留出按完整segment和最大时间缺口隔离，避免随机切分造成时间泄漏。
2. 健康随实际执行动作逐步更新，并通过$D\rightarrow\theta\rightarrow IV/功率$影响下一步可用动作。
3. 固定双堆之外加入health-greedy、Expected-max、Gamma-CVaR等强基线，并诚实降级无稳定收益方法。
4. 全段回放、容量审计、主张值和源文件哈希可追踪，负结果和不可声称边界明确。

## 4. 统计裁决

两个预声明单侧精确Wilcoxon检验的原始$p=0.00390625$，Holm校正后均为$p=0.0078125$；这不是
“双侧原始最小$p$”，但8/8段同向确实说明证据样本少且脆弱。95% segment-bootstrap区间分别为
[-0.014610982,-0.005262698]和[-0.015212701,-0.005394733]个百分点。论文必须同时报告$n=8$、
效应量、区间和多重性校正，不得用518,490个逐秒步扩大统计显著性。

## 5. 独立代理意见裁决

本地Claude Code/DeepSeek只读审稿提出三项高价值修正。容量参考敏感性和segment影响力审计
直接采纳；Expected-max/Gamma-CVaR按各自风险目标复核列为次优先级。其关于Wilcoxon最小
$p$值的表述已按实际“单侧检验加Holm校正”纠正。代理未修改任何文件。

## 6. 发表门槛与执行顺序

1. **立即完成**：35/40 kW事后诊断敏感性；单段影响力/符号翻转脆弱性；将结果接入规范表。
2. **随后完成**：检查Expected-max和Gamma-CVaR在各自目标上的收益；补跟踪容差敏感性或解释。
3. **需要用户资料**：确认额定净功率/控制器上限；取得MAT健康变量、时间戳和同源关系。
4. **正式成稿前**：确定目标期刊，完成摘要、结论、符号表、图表引用和完整参考文献核查。
5. **暂不启动**：锂电池外层和未来需求预测，直到容量与健康观测基础门槛关闭。
