# 弱退化变通试验与陈鹏小论文路线评估

更新时间：2026-07-14

## 1. 先给结论

这次最后一次退化变通已经跑通，结论是：**仅限机制演示通过，可以保留为“相对健康自适应”层，但不能称为真实退化优化或已验证的有效新方法。**

它不再猜测启停、变载、高低负载各自造成多少退化，也不使用无法由现有实车数据辨识的退化系数。控制器只知道三只虚拟堆当前位于刘占伟参数轨迹的什么位置，并在满足功率需求的候选动作中，轻微倾向于少使用健康进度更靠后的堆。

这条变通路线能够证明“在线健康程度可以进入选堆和功率分配，并改变后续状态”，但不能证明真实车辆寿命被延长。因此它适合作为陈鹏效率主线中的可选健康层或消融实验，不适合作为当前论文的唯一核心创新。

陈鹏路线本身有小论文潜力，但不能只把 OOA 换成一个新算法。建议把问题升级为：

> 面向真实负载回放的异构多堆燃料电池系统，联合求解离散运行堆选择与连续功率分配，并显式处理功率平衡、N+1、爬坡、最小驻留、故障和效率模型不确定性。

## 2. 这次变通到底怎样考虑“退化”

### 2.1 健康状态从哪里来

刘占伟的 6104 个稳定事件给出参数轨迹

```text
theta = [i0, ih, R]
```

仓库先把这条轨迹压缩成单调的相对健康进度 `h in [0, 1]`，再保存 `h -> theta(h)` 映射。这里的 `h=1` 只是刘占伟观测轨迹的末端，不是物理失效阈值，也不是 SOH=0。

每次控制前，当前 `h_j` 经过以下链路得到第 `j` 只堆在各候选电流下的可输出功率：

```text
h_j -> theta_j(h_j) -> LZW IV model -> voltage/current -> stack power
```

### 2.2 不使用退化系数，怎样在线更新

对每一个 30 分钟实车负载块，预先给定相同的全车相对健康预算 `B`。主试验取 `B=0.03`，另用 `0.01` 和 `0.06` 检查敏感性。这个数只是无量纲场景强度，不是每小时退化率。

第 `t` 步先按需求能量分配本步预算：

```text
b_t = B * (P_dem,t * dt) / sum_t(P_dem,t * dt)
```

然后按三只堆的实际输出功率份额分给各堆：

```text
delta_h_j,t = b_t * P_j,t / sum_j(P_j,t)
h_j,t+1 = h_j,t + delta_h_j,t
```

因此所有策略在同一负载块中承受完全相同的全车预算，只能改变预算落在哪只堆上，不能通过少供电来制造“少退化”。累计预算最大数值误差为 `2.220e-16`。

### 2.3 每一步怎样决策

系统是三堆 N+1，最多两堆同时运行。候选电流为：

```text
[0, 90, 120, 160, 195, 270, 370] A
```

控制器先枚举所有满足“最多两堆在线”的动作，用当前 `h` 对应的 IV 功率表预测输出。只保留跟踪误差不超过当步最优动作 `0.5 kW` 的候选，再比较氢耗、切换、爬坡和很弱的健康负载项：

```text
J = hydrogen + switch + ramp
    + lambda_h * sum_j(P_j * h_j) / sum_j(P_j)
```

`lambda_h=0` 是只做性能自适应，主健康策略取 `lambda_h=0.20`；同时检查 `0.10/0.40`。这一层是可解释的精确小规模枚举，不是假装成高级优化算法。后续若采用学习算法，枚举结果可作为小规模 oracle。

## 3. 实验怎样测，结果是否有效

需求来自 `21UBE0022` 数据整理出的连续功率块，只使用 `target_power_kw` 的负载形状，不使用中间电压突降来更新健康。2026-02 至 2026-06 共 29 个后期块用于开发回放；它们不是最终独立留出集。

实车单堆参考功率使用此前月份正功率的 99% 分位数 `45 kW`，再按双堆健康容量的 90% 映射到 `78.706 kW` 的 N+1 测试范围。结果图示例需求最高 `69.961 kW`，明确需要两堆共同供电。

三只虚拟堆初始 `h` 为 `[0.30, 0.36, 0.42]`。每个负载块都遍历这三个值在堆编号上的 6 种排列，避免结果依赖某一个幸运编号。主结果共 `29 x 6 = 174` 个配对回放。

主健康策略相对更强的“在线更新 IV、但不使用健康偏好”的性能自适应控制器，块级配对结果为：

| 指标 | 平均变化 | 块 bootstrap 95% 区间 |
|---|---:|---:|
| 最终最大 `h` | -0.012217 | [-0.014793, -0.009643] |
| 最终三堆 `h` 极差 | -0.022112 | [-0.026271, -0.017540] |
| 输出能量加权的被使用堆 `h` | -0.044099 | [-0.052481, -0.034891] |
| 不计过供电的有效电效率 | +0.0006 个百分点 | [-0.0003, +0.0017] |
| 跟踪 MAE | +0.0211 kW | [+0.0159, +0.0267] |

预算取 `0.01/0.03/0.06` 时，健康策略相对健康盲控制器的最终最大 `h` 改善分别约为 `0.00315/0.00942/0.01884`，方向一致。

但权重敏感性不是单调的：`lambda_h=0.10` 的最终最大 `h=0.433807`，反而差于健康盲控制器的 `0.429994`。这说明离散动作会造成权重阈值跳变，不能声称任意弱健康权重都稳健有效。

### 3.1 这算“有效”到什么程度

有效的是：

- 需求、选堆、功率分配、`h` 在线写回、`theta(h)` 和下一步 IV/功率能力已经闭环；
- 健康信息确实改变了哪只堆承担功率，并减小固定预算下的最大相对健康进度；
- 效果不依赖初始堆编号，也没有靠牺牲功率跟踪取得。

这里“没有牺牲”应理解为跟踪代价很小而不是完全没有：相对性能自适应，MAE 增加 `0.0211 kW`。有效电效率差的区间跨 0，不能再解释为效率改善。

不能据此声称：

- 真实退化率降低了多少；
- 启停、变载或高负载损伤被辨识；
- 车辆或电堆寿命延长；
- 三只真实电堆的 SOH 得到在线估计。

还要承认一个关键限制：健康策略的目标函数本来就偏好较小 `h` 的堆，而状态预算又按输出能量分配，所以“被使用堆更健康”是部分由设计直接带来的。当前试验主要证明链路和约束不冲突，不是一个足以单独发表的退化算法胜利。

## 4. 真正做退化感知功率分配需要什么

### 4.1 最基础：健康自适应，不辨识动作退化率

这是最容易落地的一层。至少需要：

- 每只堆稳定且不会混淆的 `stack_id`；
- 每只堆独立的电流、电压、温度和运行时间；
- 周期性的统一工况健康检查，例如固定电流点电压、IV 曲线或 EIS；
- 换堆、维修、标定、传感器更换记录；
- 能把健康观测与每只堆的历史动作按时间对齐。

有这些资料，即使仍不知道精确退化系数，也能把本项目的理想 `h` 替换成真实相对健康排序，并做“少用较老堆、保持 N+1 余量”的健康自适应控制。

### 4.2 再进一步：辨识动作与相对退化增量

除了上面的资料，还需要：

- 两次健康检查之间每只堆完整的电流、功率、启停、爬坡、温湿度和故障暴露；
- 足够多的独立时间段，使高负载、运行时间、温度和启停次数不再几乎同步增长；
- 多只堆或多辆车的重复数据，避免把日历时间和单车维护事件误认为负载效应；
- 用后期时间块验证“动作暴露模型”确实优于单纯运行时钟模型。

现有刘占伟数据没有通过这一关：618 个有效转移中，负载暴露相对运行时钟没有稳定的留出增益，累计电荷与运行时间相关系数约为 `0.965`。因此现在不能从中可靠分出高负载、变载和启停系数。

### 4.3 要谈真实寿命和物理退化

还需要：

- 台架或受控 A/B 分配试验，主动制造不同负载、启停和变载暴露；
- 重复 IV/EIS/极化曲线和一致的环境条件；
- 厂家 EOL 判据或真实维护、换堆、失效标签；
- 多堆、多车辆重复，以及至少一个完全独立的长期验证组；
- 最终最好有 HIL 或实车在线部署验证。

因此可以跳过“精确事件退化系数”，先做健康自适应；但不能跳过每只堆可追踪的健康观测，同时又声称真实退化优化。

## 5. 陈鹏原工作可以保留什么

本地资源已经确认存在：

- 论文：`G:/大论文/AI文献库/多模块燃料电池系统效率优化及功率调控策略研究_陈鹏.pdf`；
- 仿真代码：`G:/大论文/2025陈鹏/论文/仿真模型/`；
- 电堆效率拟合、空压机 MAP、Simulink 模型和部分实验数据。

可保留的核心是“每个模块的净功率/效率模型 + 多模块功率分配”。但当前代码存在几个必须先修复的问题：

- 论文第4章描述二次罚项、乘子和罚参数更新，但 `optimize_with_OOA.m` 只有 `-eta + lambda*abs(sum(P)-Pd)`，且 `lambda` 每轮乘 `0.9` 变小；可见代码不是严格的增广拉格朗日求解；
- 功率平衡容差是 `3 kW`，对效率优化过宽；
- 代码在低于 `4 kW` 时直接置零，并在末尾另做单堆覆盖，离散开关和连续分配没有统一求解；
- `cal_eff.m` 依赖 `FindSpeed`、`GetWcpBase`、`CalCmpPower` 等外部函数，当前独立目录不能直接完整复现；
- `cal_eff.m` 中空压机喘振/阻塞检查被注释，不能沿用论文中的“安全约束满足”结论；
- 空压机 114 个点在当前脚本中用于同批拟合和绘图验证，且本地重算误差与论文数字不一致，后续必须按完整转速线重新划分训练/验证/测试；
- `cal_eff` 以扣除空压机后的净功率算效率，而部分上层接口返回毛电堆功率，顶层目标可能混用了净/毛功率口径；
- 现有研究更接近逐时刻静态分配，缺少切换、爬坡、最小驻留和跨时刻状态，因此简单换算法很难形成可信创新。

## 6. 推荐的小论文问题定义

### 6.1 主线范围

当前不加入动力电池，也不做未来需求预测。输入是当步需求功率和系统状态；输出是运行堆集合及各堆功率。

```text
状态：当前需求、各堆效率模型参数、上一时刻功率、开关状态、驻留计数、故障状态
动作：离散运行堆集合 + 被选中堆的连续功率
硬约束：功率平衡、单堆上下限、最多两堆在线、爬坡、最小驻留、故障隔离
主目标：氢耗/净系统效率
次目标：切换次数、功率波动、约束违例和计算时间
可选项：本次相对健康保留项，只作消融，不解释为寿命
```

### 6.2 数据怎样用

- 陈鹏电堆与空压机数据用于建立各堆净效率曲线和系统模型；
- 三只堆先由同一基础模型加可解释参数偏差构造异质性，所有偏差必须预声明并做敏感性；
- `21UBE0022` 只提供真实需求功率形状和连续块回放；它不是多堆控制实测，也不用于证明三堆退化；
- 中间电压异常区间和未知维护边界不参与健康标定。若只使用功率需求，也要保持显式分段，不能跨边界拼成连续健康轨迹；
- 月份按时间划分训练、开发和最终测试，不能随机拆秒级样本。

### 6.3 算法选择

先建立确定性 oracle，再决定是否需要高级学习算法：

1. 小规模精确枚举或混合整数非线性/分段线性优化，作为最优性和可行性基准；
2. 陈鹏 OOA、等功率、链式加载、轮换、SQP/动态规划作为传统基线；
3. 若问题保持三堆、单步静态，精确求解已经很快，不应硬上深度强化学习；
4. 主候选应先采用运行状态图/组合搜索处理离散启停，再用边际氢耗均衡、SQP 或 MIQP 处理连续分配，并做滚动时域修正；
5. 若加入跨时刻开关、驻留、爬坡、故障和 4--6 个异构模块后，精确在线求解的 P95/P99 时间确实不能满足控制周期，才形成评估 CHDP 的实际理由；
6. 学习策略输出后仍要经过解析或优化可行域投影，保证功率平衡和硬约束，而不是只在奖励中罚违例。

CHDP 是 AAAI 2026 的参数化动作空间方法，离散策略负责模式、连续策略负责对应参数，和“选哪些堆 + 各堆给多少功率”在形式上匹配：

- <https://ojs.aaai.org/index.php/AAAI/article/view/39537>

AAAI 2025 的 action-constrained RL 工作可作为可行动作生成/约束层参考，但不能替代本系统的物理可行性审计：

- <https://ojs.aaai.org/index.php/AAAI/article/view/33714>

因此 CHDP 是二阶段候选，不是默认主算法。最稳妥的论文路线不是直接宣称“CHDP 优于 OOA”，而是：

> 用物理模型和精确求解器产生 oracle；构建带可行域保护的混合动作策略；证明其在真实负载回放、异构效率、故障和动态约束下接近 oracle，同时明显快于在线精确优化，并且零硬约束违例。

### 6.4 必须做的对比和消融

- 等功率、链式加载、轮换；
- 陈鹏 OOA；
- SQP、状态图/DP、精确枚举或混合整数 oracle；
- 一个标准混合动作 RL 基线和候选新方法；
- 去掉可行域保护、去掉异构性、去掉动态约束、去掉相对健康项；
- 正常、单堆故障、效率模型偏差和未见负载月份；
- 指标至少包括氢耗、净效率、跟踪误差、硬约束违例、切换、爬坡、运行时间和相对 oracle gap。

## 7. 最终方向判断

当前建议采用“两层主次关系”：

1. **主论文线：陈鹏效率模型 + 真实负载回放 + 有可行性和最优性基准的混合动作动态功率分配。**
2. **可选增强线：系数无关的相对健康自适应。**它说明方法能够使用在线健康程度，但不承担真实寿命主张。

只有在取得逐堆身份、周期健康检查和维护记录后，才把退化从可选增强升级为主问题。现阶段继续硬拟合启停/变载退化系数，风险明显高于它能带来的论文收益。
