# 三堆 N+1 单堆故障退出与在线重构结果

更新时间：2026-07-15

## 1. 测试到底做了什么

使用修正后的 N+1 安全随机负载，10 个留出种子均运行 `900 s`。每条轨迹在 `300 s` 以后找到第一个高负载事件，先让在线策略执行 1 秒高负载分配，再永久隔离指定电堆。

```text
故障前：三个电堆均可选，正常最多两个运行
故障当步：控制器收到 failed_stack 不可用
故障后：所有含 failed_stack 的组合立即从候选集中删除
         剩余两堆重新求解净功率分配
```

stack 2 和 stack 3 在 online hysteresis 与 instantaneous 的 `10/10` 个种子中均处于故障前在役状态。stack 1 是效率较低的备用堆，故障前不运行，因此 stack 1 故障是备用单元失效测试。

故障信息在故障当步直接提供；当前没有建故障诊断器、检测延迟或故障恢复过程。

## 2. 容量口径纠正

三个电堆数据域最大净功率为：

```text
stack 1: 54.263 kW
stack 2: 60.079 kW
stack 3: 65.807 kW
```

正常最强两堆上限是 stack 2 + stack 3 的 `125.885 kW`。真正的任一单堆故障保证容量必须取最不利剩余组合，即 stack 1 + stack 2 的 `114.342 kW`。负载最高中心和随机上限已经按后者重建，所有开发/留出结果也已重跑。

## 3. 结果

全部 `10 seeds × 3 failed stacks × 7 strategies = 210` 条故障运行满足：

- 故障当步及后续功率平衡最大绝对误差 `< 1e-8 kW`；
- 故障堆后续最大净功率为 `0 kW`；
- 重构延迟为 `0 step`；
- offline DP 在同一故障轨迹和目标下仍是事后下界。

Online hysteresis 的结果：

| 永久故障堆 | 故障后氢耗相对无故障同轨迹 | 故障当步状态变化 | 故障后 stack 1 运行占比 |
|---|---:|---:|---:|
| stack 1, 原备用堆 | 0 | 0 | 0 |
| stack 2, 原在役堆 | `+1.533% ± 0.072%` | 2.0 | 94.83% |
| stack 3, 原在役堆 | `+3.367% ± 0.150%` | 2.0 | 95.80% |

两次状态变化分别是故障堆停机和备用 stack 1 启动。stack 3 故障代价更高，因为它是三条曲线中效率最高且容量最大的电堆；被 stack 1 替代后，系统化学输入增加更多。

## 5. 这一步证明了什么

可以确认：当前离散组合和连续分配实现支持永久单堆剔除，三堆 N+1 的“备用堆接替”链路已经真正跑通。

不能据此确认：真实车辆能零延迟识别并隔离故障，也不能确认实际电堆爬坡、热状态或最小驻留允许在 1 秒内完成图中的功率转移。下一步必须把这些动态限制作为显式敏感性加入，而不是继续只看静态功率平衡。
