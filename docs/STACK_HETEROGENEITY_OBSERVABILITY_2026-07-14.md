# 三堆相对退化速率可观测性审计

更新时间：2026-07-14

## 1. 为什么现在需要这项信息

Guarded Blend在41场景参数矩阵中消除了第二边界负置信区间场景，但门控需要知道各堆相对退化
速率的排序。当前仿真中的`heterogeneity_factors`是文献压力参数，不是21UBE0022车辆上三只
真实堆的辨识结果。若没有可观测速率或可靠的不确定性区间，门控只能称离线适用域审计，不能称
已部署的在线健康决策。

## 2. 直接相关的近期方法

Tümer等在Computers & Chemical Engineering 2025文章
`Real-time optimal power sharing in multi-stack fuel cells`中，将电子传递系数`alpha`作为慢变退化
参数。其式(15)--(22)用RLS-Kalman滤波从活化损失中递推估计`1/alpha`：

- 测量端需要每只堆独立总电压；
- 回归量需要该堆电流密度和温度；
- 活化损失不是直接传感器量，还要由模型扣除开路电压、欧姆损失和浓差损失；
- 文中每5秒更新参数，每500秒更新功率分配；
- 论文明确说明估计验证使用模型生成电压，假设没有plant-model mismatch，并非真实多堆数据验证。

Gamma随机效应的EM或Bayes方法适合有多台/多堆长期退化观测后估计群体分布，不能从单只堆、
无连续身份的车辆归档中凭空恢复三只堆的相对排序。

## 3. 现有21UBE0022数据能支持什么

现有归档可提供一只总堆的DCDC输入电压、电流、冷却液进出口温度、空气进堆压力和氢气进堆
压力。它适合做单堆工况匹配后的性能残差监测，或验证observer软件接口。

当前不能支持真实三堆速率排序，原因是：

1. 没有三只堆各自独立的电压、电流和稳定`stack_id`；
2. 没有维护、换堆和控制器标定变更记录来解释2025-06、2025-08附近的时期台阶；
3. 环境校正后的长期电压趋势95%区间包含0，不能唯一归因于不可逆退化；
4. 旧MAT来自2022--2024的另一条长期链，不能与2025--2026车辆流逐行合并。

因此当前数据不能把Tümer方法直接改造成三堆在线异质性observer，也不能把文献GP随机效应分位数
称为车辆拟合值。

## 4. 需要向老师或师兄确认的数据

最低可用数据是一段同一套多堆系统的连续日志，每行至少包含：

- `timestamp`和稳定的`stack_id`；
- 每只堆独立`stack_voltage`、`stack_current`，以及有效面积或可换算的电流密度；
- 每只堆温度，或明确对应关系的冷却液进出口温度；
- 空气/氢气入口压力，最好还有流量、过量系数或阀门设定；
- 每只堆DCDC功率、开关/怠速状态和累计运行小时；
- 维护、换堆、传感器或控制器标定变更的时间与堆编号。

最好额外提供每只堆在不同日期重复进行的固定电流点或极化曲线。它们可用于把慢变退化与瞬时
温度、压力、湿度变化分离，并检验三堆速率排序是否稳定。

## 5. 数据到位后的执行顺序

1. 先做堆身份、时间连续性、维护台阶和传感器质量审计；
2. 在固定工况点复核电压残差可辨识性，不能辨识时停止；
3. 比较RLS-Kalman、EKF和简单稳健斜率，使用留出日期检验参数跟踪而非只看训练拟合；
4. 输出每只堆相对退化速率及置信区间，不只给点估计；
5. 只有排序后验概率足够高时启用Blend，否则回退fixed pair；
6. 重跑41场景参数矩阵和13个月外部负载验证。

在上述资料缺失时，当前论文应把Guarded Blend写成“在可获得相对速率估计时的安全适用域
策略”，并把真实速率observer列为尚未完成的部署条件。
