# 李俊豪双堆实车数据对FC项目的可用性审计

## 核心结论

1. 三台发动机都是 A/B 双堆串联系统：A/B 共用同一燃料电池电流，数据中没有两堆独立电流或独立 DCDC 指令。因此可审计两堆电压/功率贡献和不一致退化，但不是已有的独立堆间可控功率分配数据。
2. “目标功率”与实测 FC/DCDC 功率高度一致，更符合 FC 或 DCDC 功率指令，不符合独立整车需求功率的定义。
3. Excel 没有动力电池电压、电流或功率字段，所以不能把 `目标功率-FC功率` 当成实测电池功率，也不能从本数据严格复原原车 FC-电池分配。
4. 该数据对“双堆退化不一致性、FC工作点、氢耗、工况暴露”很有价值；对“实车需求功率→FC/电池分配”不完整。

## 对FC仓库研究路线的判断

- **车辆动力学**：有车速、里程、SOC，可派生加速度和停车工况；但没有整车质量、坡度、踏板、动力电池 V/I 和牵引电机功率，不能独立校准真实整车需求功率。
- **退化**：有长时间轴、共同电流、A/B 平均/最低/70节电压、温度、压力、空气流量、Purge，非常适合研究串联双堆的退化不一致性、弱堆识别和电压损失 proxy。
- **氢耗**：有消耗率和剩余质量、也有 FC/DCDC 功率，具备建立相对氢耗图谱的条件；绝对氢耗需先确认单位和传感器更新频率。
- **原始功率分配**：可看 FC 目标→FC原始输出→DCDC输出，可看 A/B 串联功率贡献；缺少真实整车需求和动力电池功率，不适合直接当作 FC-电池原车分配标签。
