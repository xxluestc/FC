# 最小可运行Baseline结果报告

## 数据与配置

本次实际从211,630行、46个片段的canonical重跑。预测器固定为XGBoost，历史窗口30 s，默认H=5；功率分配使用`configs/baseline.yaml`冻结权重，不进行字段消融、模型选择、权重搜索、move blocking搜索或Pareto优化。

测试功率分配段为预测测试区最长连续段的前3,600 s。63点被裁剪，占1.75%；裁剪绝对能量0.2688 kWh，最大裁剪57.56 kW。因此结果是当前可行域内baseline。

Predicted相对Constant：氢耗降低0.44%，proxy降低3.54%，电池吞吐降低1.50%，切换降低9.47%，FC总变差降低10.39%。说明因果预测相对“未来恒定”有可见作用。

Predicted相对Instant：氢耗降低0.13%、电池吞吐降低2.32%，但proxy高8.62%、切换高62.26%、FC总变差高53.35%。因此baseline尚未战胜Instant的平滑性和代理成本。

Perfect相对Constant有改善，但并未在全部指标上形成压倒性优势。原因是它仍使用有限beam、相邻档和驻留约束，且目标是多项加权和，不是每个指标分别最优。

四策略末端SOC范围0.69810–0.69877，差异很小，所以本段raw氢耗/proxy较可比；其他工况若SOC明显分离，就不能直接用raw成本判胜。

## 当前问题与后续入口

1. H=5逐秒RMSE仍高，事件F1随H下降；缺少路线、坡度和驾驶意图未来信息。
2. 车辆动力学制动段误差明显高于牵引段。
3. 氢耗是法拉第理论值，未完成实测耗氢标定。
4. degradation proxy来源跨数据链，且不是真实增量退化率。
5. 电池37 kWh、内阻和效率仍需车辆级核验。
6. baseline测试段只是一条固定连续片段，不代表所有日期/路线。

后续优化从`scripts/07--12`继续，但必须保持本报告的冻结baseline不变，新增结果应作为独立实验对照。
